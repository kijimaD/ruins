# ゲーム企画書：Ruins（仮称）

## 1. プロジェクト概要

### プロジェクト基本情報
- **タイトル**: Ruins（仮称）
- **ジャンル**: シンボルエンカウント型RPG
- **プラットフォーム**: PC（Windows/Mac/Linux）
- **開発言語**: Go言語
- **想定開発期間**: 6-8ヶ月
- **チーム構成**: 個人開発

### コンセプト
パワプロクンポケットシリーズ風のシンボルエンカウント型ダンジョンRPG。フレーバーテキストを通じて断片的に明かされる世界観と、シンプルでテンポの良い探索・戦闘を楽しむゲーム。

## 2. ゲームシステム

### 基本ゲームループ
```
探索者登録 → 装備選択 → 仲間選択 → 遺跡選択 → 探索開始 → 敵シンボルと遭遇 → 戦闘 → アイテム獲得
                ↓                    ↓
            脱出ポイントで帰還         最深部到達
                ↓                    ↓
            アイテム持ち帰り           珠獲得
                ↓                    ↓
            換金・装備購入            次の遺跡解放
                ↓                    ↓
            仲間アンロック            全遺跡クリア → エンディング
                ↓
            記録媒体を専門家に解析依頼 → システムの真実判明
```

### 探索システム
- **自動生成ダンジョン**: 不思議のダンジョン風のランダムマップ生成
- **3つの遺跡**:
  - 山の遺跡: 100階層（初級）→「火の珠」
  - 海底遺跡: 100階層（中級）→「水の珠」
  - 樹海遺跡: 100階層（上級）→「風の珠」
- **階層システム**:
  - 各遺跡は100階層で構成
  - 5階ごとに脱出ゲート（帰還地点）を設置
  - 20階ごとにボス部屋を配置
  - 20階クリアするごとにそこから始められるようになる
  - 階が深まるごとに難易度が上昇していく
- **特殊部屋**: ランダム。20階層につき1-2個の固定イベント部屋
  - 見た目が異なる（特別なタイル表示）
  - 環境記録を調査可能
  - 階層ごとに内容は固定、配置はランダム
  - 一部の部屋では自動整備システムの劣化により、システム情報が漏洩している
- **階層ボス**: 20階ごとに守護者ボスが待ち受ける
  - 各遺跡に5体のボス（20階、40階、60階、80階、100階）
  - 最深部（100階）のボスが最も強力
- **宝箱**: 各階層にアイテム入り宝箱を配置
  - 配置されるアイテムは探索者をおびき寄せるための報酬
  - その時代の技術レベルに合わせて違和感のないものを選択
  - 現在の第7次文明には戦後復興期レベルの装備を配置
- **珠**: 各遺跡の最深部（100階）で入手できる神秘的な宝石
- **遺跡間移動**: 不思議な転送装置で各遺跡を行き来
- **脱出ゲート**: 5階ごとに設置された帰還地点
  - 脱出ゲートから帰還すると探索終了
  - 入手したアイテムを持ち帰ることができる
  - 途中で倒れた場合、アイテムは失われる
- **遺跡の危険性**:
  - 下層階ほど敵が強力になり、死亡率が急激に上昇
  - 金目的の探索者は数階層で撤退するため、システムの測定には役立たない
  - 深部到達には強い感情的動機が必要

### 戦闘システム
- **ターン制コマンドバトル**: 攻撃・魔法・アイテム・逃走
- **パーティ編成**: 最大4人のパーティ（主人公以外は自動行動）
- **属性**: 物理・火・電・酸・光
- **仲間の特徴**: 各キャラクター固有の戦闘スタイル

### 成長システム
- **レベルアップ**: 経験値を獲得してレベル上昇（探索終了時にリセット）
- **装備**: 武器・防具・アクセサリの3種類（持ち込み可能）
- **スキル**: レベルに応じて新スキル習得
- **アイテム持ち込み制限**: 消耗品は持ち込み不可、装備のみ持ち込み可能
- **換金システム**: 持ち帰った換金アイテムで新装備・仲間をアンロック
- **仲間アンロック**: 換金で得た資金で新たな仲間を雇用可能
- **深層攻略の条件**:
  - アイテム収集による装備の充実
  - 換金による装備の整備とアップグレード
  - スキルレベルの向上
  - 20階区切りでの段階的な攻略
  - 上記なしでは深い階層の攻略は困難

## 3. 世界観・ストーリーテリング

### 世界観（ゲーム内では明示しない）
- **時代**: 昭和25年頃の日本（に見せかけた遠い未来）
- **舞台**: 地下深くに眠る古代遺跡
- **技術**: 戦後復興期の技術水準（に見せかけた高度文明の痕跡）
- **謎**: 繰り返される文明の興亡
- **システムの正体**: 文明進行度管理システム - 一定レベルに達した文明を強制的に原始時代まで後退させる

### ストーリーテリング手法
**環境ストーリーテリング**: 断片的な情報から世界観を推測する探索型物語
- 直接的なストーリーイベントは最小限（開始・珠獲得・終了のみ）
- アイテム説明文、敵図鑑、環境テキストで世界観を表現
- プレイヤーの想像力と考察を促す設計
- 具体的な年代や地名は意図的に曖昧にする

### 最小限のストーリーイベント
1. **ゲーム開始**:
  - 登録証「探索者登録番号 2847 17歳 男性。目的: 遺跡探索および『珠』の回収
  - 診断書「□□(49歳 女性) 原因不明の昏睡」
  - メモ「古い言い伝えによると、珠には不治の病を治す力がある」
2. **珠獲得時**: 「珠を回収。必要な素材があと2つ。」
3. **エンディング**: 「全ての珠が揃った。さあ、選べ。」

### 主人公情報の段階的開示
**段階1: システムメッセージ（開始時）**
- 「母の治療」という動機の提示
- 抽象的な情報のみ
- 例：「備考: 母の治療費は組合が負担する」

**段階2: アイテム発見（中盤）**
- 個人的な所持品の発見
- 具体的な名前や関係性の判明
- 例：「血染めの写真」「薬のメモ」で母の名前と病名が判明

**段階3: 記録発見（終盤）**
- 探索者記録での真実への言及
- システムの視点からの主人公像
- 例：「探索者記録 - 第2847号」で文明管理システムの一端が明らかに

**段階4: 環境反応（全体）**
- 特定のアイテムや場所での感情的反応
- プレイヤーの推測を促す表現
- 例：「なぜか胸が締め付けられる」「無性に急がなければ」

## 8. フレーバーテキスト設計指針

### テキスト作成ルール
1. **断片的**: 完全な説明はせず、ヒントのみ提供
2. **具体的**: 抽象的な表現より具体的な描写
3. **感情的**: プレイヤーの感情に訴える要素
4. **一貫性**: 世界観に矛盾しない内容

### カテゴリ別方針
- **アイテム**: 過去の使用者の痕跡、不可解な機能、主人公の個人的な所持品
  - **換金アイテム**: 古代文明の日常品、給与や記録など生活の痕跡
  - **装備アイテム**: 実験場の職員が使用していた道具、警備用装備
  - **消耗品**: 実験用の薬品、被験体用の補給品
  - **特殊アイテム**: 主人公の過去に関連する重要な手がかり
  - **戦争関連アイテム**: 太平洋戦争時代の遺物、軍需品、配給制度の痕跡
  - **文化アイテム**: 戦後復興期の娯楽、生活様式、復興期の文化を示すもの
- **敵**: 元は人間だった示唆、哀れみを誘う要素
- **環境**: 文明の痕跡、システムの存在を示唆、主人公の実験記録
  - **戦争関連環境**: 防空壕、軍需工場、配給所など戦時体制の痕跡
  - **文化環境**: 映画館、学校、ラジオ局など戦後復興期の文化施設
- **座標・数値**: 現実を想起させるが確定はさせない
- **感情表現**: 主人公の内面的な反応、推測を促す曖昧な表現

### 内部設定との関連
- 遺跡名は実際の地理を暗示（山→富士、海底→東京湾、樹海→屋久島）
- 座標表示は実在の位置だが、プレイヤーには明示しない
- 「第7次」「文明進行度68%」など、世界観の核心に触れる数値を散りばめる
- 戦後復興期の技術水準に見せかけて、実際は2万年後の文明の痕跡

### 文明管理システムの詳細（内部設定）
- **基本方針**: 人類文明を自由に発展させながら、一定レベルに達したら強制的にリセット
- **リセット基準**: 文明が遺跡を踏破できるレベルの技術力に到達した時点
- **システム目的**: 文明の無制限な発展による破滅的結果（核戦争、環境破壊等）を防ぐ
- **サイクル管理**: 現在は第7次文明、これまで6回のリセットを実行済み
- **監視方法**: 各地の遺跡が文明進行度の測定装置として機能
- **リセット手段**: 技術的後退、記録の消去、人口調整等により原始時代レベルまで文明を後退
- **進行度表示**: 「文明進行度68%」は現在の技術発展レベルを示す
- **リセット条件**: 進行度が100%に達すると自動的に文明リセットが実行される
- **主人公の位置**: 第7次文明の末期に生きる一般人、システムの存在は知らない
- **システムの劣化**: 2万年間の運用により、初期システムにほころびが目立ち始めている

### 探索動機システム（内部設定）
- **病気の散布**: システムが意図的に原因不明の病気を散布し、遺跡探索の動機を作る
- **伝承の流布**: 「遺跡の珠に治癒力がある」という伝承を広めて探索者を誘導
- **技術測定**: 遺跡を踏破する技術力に達した文明かどうかを測定するための仕組み
- **MK-47症候群**: システムが散布した病気の一種、治療法は意図的に隠されている
- **探索者選別**: 病気を患う家族を持つ者を優先的に探索者として選別
- **データ収集**: 探索者の行動パターンから文明の技術レベルを分析
- **遺跡の危険性**: 遺跡探索は極めて危険で、金目的の探索者は低層で撤退する
- **動機の強度**: 深層まで到達するには強い感情的動機（家族の命）が必要
- **自然選別**: 金銭目的の探索者は自然に排除され、真に必要な者のみが深部に到達

### システムの劣化要因（内部設定）
- **長期間運用**: 2万年間の継続運用により、初期設計にない問題が蓄積
- **想定外の進化**: 7回のリセットを通じて、人間が予想外の進化を遂げている
- **保守機能の低下**: システムを保守する機能自体が経年劣化している
- **データ蓄積エラー**: 膨大なデータの蓄積により、処理能力に限界が生じている
- **感情的動機の誤算**: 強い感情的動機が、システムの想定を超える力を発揮する
- **遺跡守護者の弱体化**: 長年の稼働により、遺跡の守護システムが劣化している
- **予期せぬ変数**: 主人公のような個体が、システムの計算を狂わせる可能性

### 報酬アイテム配置システム（内部設定）
- **時代適応配置**: 各文明の技術レベルに合わせて、違和感のないアイテムを配置
- **おびき寄せ機能**: 探索者を深部まで誘導するための報酬として機能
- **技術レベル調整**: 現在の文明（第7次）に適した戦後復興期レベルの装備を配置
- **価値観の利用**: その時代の人々が価値を感じるアイテムを選択して配置
- **進歩の測定**: 探索者がどこまで進めるかで、文明の技術進歩を測定

### 情報漏洩システム（内部設定）
- **自動整備システム**: 探索済みフロアをシステムが自動的に修復・調整・機械整備
- **処理能力の限界**: 2万年の運用でシステムの処理能力が限界に達し、完全な情報消去ができない
- **ワープ制限**: 探索済みフロアには他の探索者をワープさせず、自動整備を実施
- **システム劣化**: 長期運用による自己修復機能の劣化で、記録媒体の完全破棄に失敗
- **専門家による解析**: 探索者が持ち帰った記録媒体を専門家が解析し、真実が判明

### 遺跡に対する社会認識（内部設定）
- **一般認識**: 遺跡は「システム」ではなく、古代からある謎の危険な場所として認識
- **社会の関心**: 近年の科学進展、発明、戦争、文化が流行の話題で、古代遺跡は忘れ去られがち
- **関心を持つ層**: 研究者、変わり者、伝承を求める人、誰かを助けようとする人のみが関心
- **伝承の扱い**: 「珠に治癒力がある」という話は迷信・民間療法の一種として扱われている
- **探索者の位置**: 社会の周縁にいる人々で、一般的には理解されない存在

### 地上に複数ある遺跡入り口（ゲーム内では明示しない）
- **主人公が使用する入り口**: 富士山麓の「魔穴（まあな）」
  - 地元では「呪われた穴」「祭り穴」として恐れられている
  - 自然の洞窟として認識され、古くから存在
  - 行方不明者が多発する危険地帯
  - 探索者組合が公式な探索地点として管理

- **その他の入り口候補**:
  - 東京湾の小島（廃墟、立入禁止）
  - 奥多摩の廃坑道（明治時代の鉱山跡）
  - 伊豆の海蝕洞（潮の満ち引きで出現）
  - 房総半島の古墳群（古代の祭祀場跡）

- **共通認識**:
  - 各入り口は独立した危険地帯として認識
  - 地下で繋がっていることは人類には知られていない
  - 「システム」という認識はなく、古代遺跡として恐れられる
