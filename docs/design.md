# RUINS - ゲーム企画書

## エグゼクティブサマリー

**RUINS**は、廃墟世界を舞台にした1人称ローグライクRPGです。プレイヤーは危険なダンジョンを探索し、アイテム収集・クラフト・戦略的な選択を通じて成長していきます。

### 企画の核心
- **実在感ある世界観**: フレーバーテキストによる深い没入体験
- **意味のある選択**: 数値最適化を超えた感情的な判断システム
- **学習と成長**: パーマデスでありながら、プレイヤー自身の成長を実感できる設計

### ターゲット・市場性
- **メインターゲット**: ローグライク愛好者、世界観重視のRPGプレイヤー
- **サブターゲット**: レトロゲーム愛好者、ブラウザゲームユーザー
- **プラットフォーム**: Web（WebAssembly）、PC
- **想定プレイ時間**: 1セッション30分〜1時間、全体10〜20時間以上

## ゲーム概要

### 基本コンセプト
廃墟となった世界で、プレイヤーは探索者として危険なダンジョンに挑みます。ターン制での戦略的な探索、リソース管理、そして豊富なクラフトシステムが特徴です。

### ジャンル・特徴
- **ジャンル**: ローグライクRPG（ターンベース）
- **視点**: 1人称レイキャスティング（3D風2D）
- **探索**: グリッドベース・ターン制移動
- **戦闘**: 同一マップ上でのターンベース統合戦闘
- **成長**: 装備・アイテム依存、経験値なし
- **言語**: 完全日本語対応

- **装備重量影響**: 重装備による移動・攻撃コスト増加

- **リアリズム**: 行動には実際に時間がかかるという自然な表現
- **バランス調整**: 強力な行動ほど隙が大きくなる仕組み

**採用する要素:**
- **世界観構築**: シリアスな背景ストーリーと軽快なゲームプレイの両立

- **選択の重み**: 通貨の使い道が戦略の核心

- **回収ミッション**: 死亡地点への回収が新たなゲームプレイを生む
- **学習曲線**: 失敗から通貨管理の重要性を学ぶ

## 核心的な面白さ

### プレイヤー体験の設計

#### 1. 「実在感ある世界での探索」
RUINSの最大の差別化要素は、世界観とテキストによる深い没入感です。

**実現方法:**
- 各アイテム・エリアに込められたフレーバーテキスト
- 廃墟世界の背景ストーリーを感じられる細やかな描写
- 単なる数値ではない、意味と歴史を持つオブジェクトとしての体験
- テキストを読むこと自体が探索の楽しみになる設計

#### 2. 「意味のある選択とその結果」
数値最適化を超えた、感情的な判断を伴う選択システム。

**具体的なジレンマ:**
- **重量管理**: 貴重品（高価格・重い） vs 実用品（便利・軽い）
- **リスク判断**: HP残量 vs 未探索エリアの誘惑
- **戦闘選択**: 逃走（安全） vs 迎撃（リスク・報酬）
- **探索継続**: 5階到達時の「あと1階だけ...」の心理的プレッシャー

#### 3. 「プレイヤー成長による体験質の向上」
パーマデスでありながら、継続的な成長を実感できるシステム。

**二重成長システム:**
- **知識の成長**: 失敗からの学習が次回プレイの戦略につながる
- **装備の蓄積**: ステータスリセット + 装備持ち越しによる新しい進歩感
- **愛着の形成**: 苦労して手に入れた装備が次回冒険の支えになる

### プレイヤー感情の設計
- **緊張感**: リスク判断、重量管理、冒険続行判断での心理的プレッシャー
- **達成感**: 試行錯誤による成功体験、装備持ち越しによる蓄積感
- **発見の喜び**: フレーバーテキストによる世界観発見、新アイテムの特殊効果発見

## ゲームシステム

### コアループ
1. **ステージ選択**: 3つのランダムな選択肢から次のステージを選ぶ
2. **燃料管理**: 燃料残量を確認、燃料補給の必要性を判断
3. **探索**: グリッド移動でのダンジョン探索、世界観体験
4. **発見**: 新エリア・素材・アイテム・燃料発見、背景ストーリーの発見
5. **戦闘**: 敵との戦闘、通貨獲得、燃料消費
6. **判断**: 戦闘選択、リソース管理（燃料/HP/通貨/重量/アイテム）
7. **通貨使用**: 燃料購入、装備購入、強化の選択
8. **収集**: 戦利品・素材・燃料収集、持ち帰り品の選択と取捨判断
9. **製作**: クラフトによる装備強化、次回探索への準備
10. **リスク判断**: 冒険続行か撤退か、燃料は足りるか、通貨を使うか貯めるかの戦略的決断

### プレイヤー体験のフロー
```
【探索開始】 → 燃料100%で出発
【ステージ選択】 → 3つの選択肢から戦略的判断
【燃料確認】 → 「燃料は足りるか？」の不安
【探索中】 → 実在感ある世界へ没入、燃料が徐々に減少
【フレーバーテキスト体験】 → 新しい発見への期待
【戦闘・通貨獲得】 → 通貨が貯まる喜び、燃料消費の焦り
【燃料ルートの選択】 → 「今回は燃料ルートが出るか？」の緊張
【通貨使用の判断】 → 「燃料を買うか、装備を買うか」の葛藤
【燃料枯渇の恐怖】 → 「次の選択肢に燃料ルートが出なければ...」
【成功/失敗からの学習】 → 燃料・通貨管理の重要性を学ぶ
【死亡・通貨ドロップ】 → 回収ミッション、燃料消費のジレンマ
【装備持ち越しでの再挑戦】 → 継続的な成長体験、次は燃料管理を改善
```

### ステージ選択システム

ダンジョン探索において、各ステージクリア時にプレイヤーは次に進むステージを選択します。この選択システムは、リスクとリターンのバランスと、限られた選択肢からの戦略的判断を求める核心的なゲームプレイ要素です。

#### 選択の仕組み

**基本メカニズム:**
- ステージクリア時に、選択肢プール（全7種類）から**ランダムに3つ**が提示される
- プレイヤーは提示された3つの中から1つを選択し、次のステージへ進む
- **重複なし**: 同じ選択肢が複数出ることはない
- **ランダム性**: 必要な選択肢が出ないこともある（例：燃料が必要なのに燃料ルートが出ない）
- 各選択肢は、トレードオフが明確に設計されている

**戦略的要素:**
- 「今必要なもの」vs「今出ている選択肢」の判断
- 燃料が少ない状態で燃料ルートが出ない緊張感
- 「このチャンスを逃すと次はいつ来るか分からない」という判断
- リスクの高い選択肢しか出ない状況での決断
- 現在のリソース状況（燃料/HP/装備/通貨）に応じた優先順位付け

#### 選択肢プール（全7種類）

各ステージクリア時に、以下の7種類から重複なしでランダムに3つが提示されます。

**1. 燃料補給ルート（Fuel Supply）**
- **リターン**: 燃料を確実に入手、燃料消費量も低い
- **リスク**: 敵は中程度、高価値品は少ない
- **テーマ例**: 補給所跡、食料倉庫、農場廃墟
- **戦略**: 燃料が少ない時の安定選択、燃料管理の保険

**2. 生存重視ルート（Safe Passage）**
- **リターン**: 敵が少ない/弱い、燃料消費が少ない、HP回復ポイントあり
- **リスク**: 報酬が少ない（通貨・素材ともに低い）
- **テーマ例**: 避難経路、整備された通路、放棄された安全区画
- **戦略**: HPが少ない時、リソースを温存したい時

**3. 価値重視ルート（High Value）**
- **リターン**: 高価値品・レア素材・大量通貨
- **リスク**: 敵が強い、燃料消費が大きい、トラップ多数
- **テーマ例**: 宝物庫、研究所、金庫室
- **戦略**: 装備が整っている時、ハイリスク・ハイリターン狙い

**4. 装備強化ルート（Equipment Focus）**
- **リターン**: 武器・防具素材、装備品、強化キット
- **リスク**: 燃料補給が少ない、敵は中〜強程度
- **テーマ例**: 武器庫、工房、装備保管所
- **戦略**: 装備を強化したい時、戦闘力向上を優先

**5. 情報収集ルート（Intelligence）**
- **リターン**: 次回の選択肢が事前に分かる、マップ情報、敵の弱点データ
- **リスク**: 即座の報酬は少ない、燃料は普通
- **テーマ例**: 司令室、データセンター、監視所
- **戦略**: 長期的な戦略を立てたい時、次の選択に備える

**6. 速攻ルート（Quick Pass）**
- **リターン**: ステージが短い、燃料消費が非常に少ない
- **リスク**: 報酬がほぼない、敵を避けて進む必要あり
- **テーマ例**: 緊急脱出路、エレベーター、ショートカット
- **戦略**: 燃料節約最優先、深層到達を急ぐ時

**7. 挑戦ルート（Challenge）**
- **リターン**: 超高報酬、レア装備、大量通貨、特別なレシピ
- **リスク**: 強敵・ボス級、燃料大量消費、高難度
- **テーマ例**: ボス部屋、危険区画、封鎖エリア
- **戦略**: 完全に準備が整った時、最高報酬を狙う時

#### トレードオフマトリクス

各選択肢の特性を可視化：

| ルート | 燃料 | 安全性 | 報酬 | 時間効率 | 特殊要素 |
|--------|------|--------|------|----------|----------|
| 燃料補給 | ⭐⭐⭐ | ⭐⭐ | ⭐ | ⭐⭐ | - |
| 生存重視 | ⭐⭐ | ⭐⭐⭐ | ⭐ | ⭐⭐⭐ | HP回復 |
| 価値重視 | ⭐ | ⭐ | ⭐⭐⭐ | ⭐ | - |
| 装備強化 | ⭐ | ⭐⭐ | ⭐⭐ | ⭐⭐ | 装備素材 |
| 情報収集 | ⭐⭐ | ⭐⭐ | ⭐ | ⭐⭐ | 先読み |
| 速攻 | ⭐⭐⭐ | ⭐⭐ | ☆ | ⭐⭐⭐ | 時短 |
| 挑戦 | ☆ | ☆ | ⭐⭐⭐ | ☆ | レア品 |

⭐⭐⭐ = 非常に良い、⭐⭐ = 良い、⭐ = 普通、☆ = 悪い

#### サブテーマシステム

各選択肢には、ランダムに選ばれる「サブテーマ」が存在します。サブテーマは選択画面では表示されず、ステージに入ってから判明します。

**サブテーマの効果:**
- **マップ生成**: 部屋の配置・通路の構造・広さが変化
- **敵の配置**: 出現する敵の種類・密度が微妙に変化
- **アイテムテーブル**: ドロップするアイテムの傾向が変化
- **特殊ギミック**: トラップ・環境効果の有無
- **環境効果**: 移動コスト、視界制限、状態異常などの追加要素

**サブテーマの例:**

**燃料補給ルートのサブテーマ:**
- **食料倉庫**: 大量の食料、敵は少ない、ネズミ系小型敵
- **農場廃墟**: 中量の食料、野生生物敵、自然素材も入手可
- **補給基地**: 燃料+弾薬、中程度の敵、軍事素材あり

**価値重視ルートのサブテーマ:**
- **宝物庫**: 通貨大量、強力なガーディアン、トラップ多数
- **研究所**: レア装備、改造生物、放射能エリア
- **金庫室**: 高価値品、セキュリティロボット、パズル要素

**生存重視ルートのサブテーマ:**
- **避難経路**: 敵ほぼなし、燃料少量、医療品あり
- **整備通路**: 弱い敵、ショートカット、照明良好
- **安全区画**: HP回復ポイント、補給品、休憩可能

**挑戦ルートのサブテーマ:**
- **ボス部屋**: 強力なボス1体、超高報酬、特別なレシピ
- **危険区画**: 強敵多数、レア素材大量、環境ダメージ
- **封鎖エリア**: 未知の敵、最高級装備、高難度ギミック

#### ゲームデザイン上の意義

**1. リプレイ性の向上**
- 7種類から3つのランダム選択により、毎回異なる選択肢が出現
- サブテーマの組み合わせで同じルートでも体験が変化
- プレイヤーの選択と運により、無数のルート組み合わせが生まれる

**2. 戦略的深度の増加**
- 「今出ている選択肢」と「今必要なもの」のギャップ管理
- 複数リソース（燃料/HP/装備/通貨）の優先順位判断
- 次に必要な選択肢が出ないリスクを考慮した先読み戦略
- 情報収集ルートによる次回選択肢の事前把握

**3. リスク・リターンの明確化**
- トレードオフマトリクスによる選択結果の予測可能性
- ランダム性により完璧な計画は不可能（ローグライクの本質）
- 現在の状況（燃料残量/HP/装備状態）に応じたリスク管理

**4. 緊張感の創出**
- **燃料枯渇の恐怖**: 燃料が少ないのに燃料ルートが出ない状況
- **選択の重み**: 「このチャンスを逃すと次はいつ来るか」という判断
- **ギャンブル性**: リスクの高い選択肢しか出ない状況での決断
- **計画の崩壊**: 想定していた選択肢が出ず、戦略変更を強いられる

**5. 学習曲線の形成**
- 各ルートの特性を理解していく過程
- 「どの状況でどのルートを選ぶべきか」の経験則構築
- サブテーマのパターン学習による予測精度向上
- 失敗から次回への戦略を構築

**6. 感情的な選択の創出**
- 「燃料が少ないが、価値重視に賭けるべきか」の葛藤
- 「HP低いが生存ルートが出ない、どうするか」の緊張
- 「挑戦ルートが出た、今が挑むタイミングか」の判断
- 「情報収集で安全策か、即座の利益を取るか」の選択

### 燃料システム

探索を続けるために必要不可欠なリソースとして、「燃料」を導入します。燃料は時間経過で消費され、0になるとゲームオーバーとなります。

#### 基本メカニズム

**燃料の消費:**
- **ステージクリア時**: 各ステージクリア時に一定量消費（ルートにより変動）
- **ターン経過**: 探索中のターンごとに少量消費
- **戦闘**: 戦闘ターンでは通常より多く消費
- **アクション**: 重い行動（装備変更等）で追加消費

**燃料の回復:**
- **燃料補給ルート**: 確実に燃料を入手できる
- **アイテム発見**: 探索中に燃料アイテムを発見
- **敵ドロップ**: 一部の敵が燃料をドロップ（低確率）
- **休憩ポイント**: 生存ルートのHP回復時に少量回復

#### 燃料残量による影響

**残量80-100%: 安全圏**
- すべての選択肢を自由に選べる
- 挑戦ルートや価値重視ルートへの挑戦が可能
- 戦略的な選択を楽しめる段階

**残量50-80%: 注意段階**
- 燃料消費の多いルートを避け始める
- 燃料補給ルートが出た時の価値が上がる
- リスク管理を意識し始める段階

**残量20-50%: 危険段階**
- 燃料補給ルートが最優先
- 燃料ルートが出ないと焦りが生まれる
- 速攻ルートで燃料節約を考慮
- 戦闘回避を重視し始める

**残量0-20%: 緊急事態**
- 燃料補給ルートが出なければ詰み寸前
- すべての判断が燃料優先になる
- 高い緊張感とギャンブル性
- 生存ルートや速攻ルートで燃料を節約

**残量0%: ゲームオーバー**
- 探索不能、即座に死亡
- 通貨と装備の一部をロスト

#### ゲームデザイン上の意義

**1. タイムプレッシャーの創出**
- 燃料という時間制限により「永遠に探索」はできない
- 深層到達には燃料管理が必須
- 「もう少しだけ」という判断に燃料残量が影響

**2. ステージ選択への影響**
- 燃料残量が選択肢の優先順位を変える
- 「燃料ルートが出ない」という状況が緊張感を生む
- 燃料消費の多いルートを選ぶリスクが明確

**3. 探索スタイルの多様化**
- 速攻型: 速攻ルート優先、報酬は少ないが深層到達
- バランス型: 燃料管理しながら報酬も狙う
- リスク型: 燃料を気にせず価値重視、短期決戦

**4. Dark Souls式通貨システムとの相互作用**
- 通貨を使うべきタイミング（燃料買い vs 装備強化）
- 燃料不足時は死亡リスク増、通貨ロストの恐怖が増す
- 「燃料を買うか、装備を買うか」の戦略的判断

**5. リソース管理の深化**
- 燃料/HP/通貨/装備の4つのリソースを同時管理
- どれを優先するかの状況判断
- 複数リソースのトレードオフによる戦略性

### ターンベース探索システム

#### 基本仕様
- **グリッド移動**: 1マス単位での移動
- **アクションコスト**: 全行動に時間コストを設定
- **ターン制**: プレイヤー行動→敵行動の繰り返し
- **統合戦闘**: 探索マップ上で直接戦闘実行
- **レイキャスト表示**: 1人称視点は維持

#### アクションコストシステム（CDDA準拠）

CDDAのAction Points（AP）システムを参考にした精密な行動コスト管理。

**基本メカニズム:**
- **APの獲得**: 毎ターン開始時に「敏捷性」値分のAPを獲得
- **基本移動コスト**: 通常地形での1マス移動 = 100AP
- **ターン終了条件**: APが0以下になるとターン終了

**行動コスト例:**
```
基本行動:
- 移動（平地）: 100AP
- 移動（対角）: 142AP（√2倍）
- 通常攻撃: 100AP
- 重武器攻撃: 150-300AP
- アイテム使用: 80-120AP
- 装備変更: 200AP
- スキル使用: 120-400AP
```

**CDDA式債務システム:**
APが不足する行動を行った場合の処理例：
```
例：敏捷性100、重武器攻撃250APの場合

ターン1:
- 攻撃実行（250AP消費）
- 残りAP: 100 - 250 = -150
- ターン終了（APが負の値）

ターン2:
- 敏捷性分AP獲得: +100
- 現在AP: -150 + 100 = -50（まだ負）
- 即座にターン終了

ターン3:
- 敏捷性分AP獲得: +100
- 現在AP: -50 + 100 = +50
- 行動可能（50APで行動開始）
```

**コスト調整要素:**
- **地形**: 難地形での移動コスト増加（茂み150AP、沼地200APなど）
- **装備重量**: 重装備時の移動・攻撃コスト増加
- **状態異常**: 毒、疲労、負傷による速度低下
- **スキル熟練**: 慣れた武器・魔法の使用コスト減少

**戦略的要素:**
- **タイミング戦略**: 重い攻撃は相手の行動前に実行するのが有効
- **AP管理**: 移動とアクションのバランス調整
- **継続行動**: 長時間アクション（クラフト等）の中断・再開システム

#### 特徴

- **スキル選択**: 多様な攻撃手段をターン内で選択
- **レンジ戦闘**: 遠距離・近距離攻撃を距離に応じて実装
- **位置戦略**: マップの地形・障害物を活用

#### 攻撃タイプ
- **近接攻撃**: 隣接マスの敵を攻撃
- **遠距離攻撃**: 射線上の敵を攻撃（射程制限あり）
- **範囲攻撃**: 複数マスの敵に同時攻撃

### アイテム・クラフトシステム

#### アイテム構成
- **攻撃スキル**: 54種類（近接、銃器、魔法、範囲攻撃）
- **防具**: 30種類（頭部、胴体、脚部、装飾品）

- **クラフトレシピ**: 75種類
- **素材**: 24種類

#### 購入と合成の棲み分け

**基本方針:**
- **購入**: 安定した基礎固め、保険的役割
- **合成**: ガチャ要素、やり込み要素

**購入システム（店での購入）**
- **確実性**: 必要な装備を確実に入手できる
- **初期装備**: ゲーム序盤の基本装備供給源
- **補完的役割**: 足りない部位を埋める調整用、日用品の調達用
- **価格**: 基本価値の2倍（高コスト）
- **性能**: 基本性能が低め・安定
- **世界観**: 地上の技術で作られた普通の装備

**合成システム（クラフト）**
- **やり込み要素**: 高性能装備を狙う目標
- **素材収集の動機**: ダンジョン探索の目的を増やす
- **ランダム性**: 性能にランダム補正（-10～+9等）
  - 最低値だと購入品を下回る
  - 最高値なら購入品を大きく上回る
- **コスト**: 素材収集に時間がかかる
- **世界観**: 古代の合成機械で作られた特殊装備、地髄エネルギーによる物質変換

**段階的な役割変化:**

```
【序盤(1-20階)】
購入: 基本装備を揃える主要手段
合成: 素材が少なく、あまり利用できない

【中盤(20-60階)】
購入: 不足部位を補う補助的役割
合成: 高性能装備を狙って本格的に利用開始

【終盤(60-100階)】
購入: ほぼ使わない（性能が低い）
合成: 最深層攻略に必要な高性能装備の主要入手手段
```

**設計意図:**
- 両システムが補完的に機能し、プレイの選択肢を増やす
- 購入は「確実性」、合成は「ギャンブル性」という明確な差別化
- ゲーム進行に応じて自然に合成へシフトする設計
- 素材収集がダンジョン探索の動機づけになる
- 共通の通貨（CZ）を使うことで、「何に使うか」の戦略的選択が生まれる

- インベントリ圧迫時の「何を捨てるか」判断

**基本コンセプト:**

#### 通貨の用途

通貨は多目的リソースとして機能：

1. **アイテム購入**: 商人NPCからの購入

   - 装備品
   - 素材

2. **装備強化**: 武器・防具の強化

#### 死亡時のメカニクス

**Dark Souls式のリスク・リターンシステム:**

```
【死亡時】
├─ 所持通貨を全てその場に落とす
├─ プレイヤーは通貨0で復活
└─ 死亡地点にマーカー表示

【回収フェーズ】
├─ 死亡地点まで戻れば全額回収可能
├─ 回収前に再度死亡 → 前回の通貨は完全消失
└─ フロア移動で回収不可になる可能性
```

**心理的緊張感の創出:**
- 大量の通貨を持つほど死亡リスクが重くなる
- 「いま使うべきか、貯めるべきか」の判断
- 「回収に戻るか、諦めて先に進むか」の選択

### ターンシステム実装詳細

#### CDDAベースのAPシステム実装
```go
type ActionPoints struct {
    Current int // 現在のAP（負の値も可能）
    Speed   int // 基本Speed値（敏捷性）
}

type ActionCost struct {
    Move       int // 移動コスト
    Attack     int // 攻撃コスト
    Item       int // アイテム使用コスト
    Equipment  int // 装備変更コスト
}

// ターン開始時のAP回復
func (ap *ActionPoints) StartTurn() {
    ap.Current += ap.Speed
}

// アクション実行とAP消費
func (ap *ActionPoints) ConsumeAP(cost int) bool {
    ap.Current -= cost
    return ap.Current > 0 // false = ターン終了
}
```

#### 債務システムの実装例
```go
// 重い行動（攻撃等）の処理例
func executeHeavyAction(player *Player, actionCost int) {
    // アクションを実行（APが足りなくても）
    executeAction()

    // APを消費（負の値になることがある）
    player.AP.ConsumeAP(actionCost)

    // APが負の場合、ターン終了
    if player.AP.Current <= 0 {
        endPlayerTurn()
        // 次のターンでAP回復まで待機
    }
}
```

#### 地形・装備によるコスト調整
```go
type TerrainType int

const (
    Plain  TerrainType = 100  // 平地
    Bush   TerrainType = 150  // 茂み
    Swamp  TerrainType = 200  // 沼地
    Wall   TerrainType = -1   // 通行不可
)

func calculateMoveCost(terrain TerrainType, equipment *Equipment) int {
    baseCost := int(terrain)

    // 装備による修正
    if equipment.IsHeavy() {
        baseCost += 50
    }

    // 対角移動の場合は1.42倍
    if isDiagonalMove {
        baseCost = int(float64(baseCost) * 1.42)
    }

    return baseCost
}
```
